## 講義契約

`course_2/chapters/<note>.md` 是一份投影片對應的唯一正典講義。Phase 1 建立它；Phase 2、3、4、5、6、7 都是**就地修改同一個檔案**。永遠不要分裂成 `phase2/`、`*-practice.md` 或第二份競爭的 Markdown。

**講義檔內不得出現 phase 進度**——frontmatter 和內文都不行。講義會合併成一份列印，任何非教學內容都會印出去。進度唯一記錄處是 `course_2/進度.md`。frontmatter 只保留不隨進度變動的識別欄位：

```yaml
---
chapter: "12"
slug: chi-square-tests
title: 卡方檢定
source_deck: Quan1150(Chi-Square).pdf
---
```

## 講義結構

除非教學上明顯需要調整，每份講義依此順序（括號註明由哪個 phase 填入）：

1. 章節標題
2. 先備知識（Phase 3：course_1 對應章節的白話摘要與相對路徑連結；無對應則說明本章自給自足）
3. 學習目標（Phase 1：學完能做什麼判斷或計算，用可觀察的動作描述）
4. 本章重點一覽（Phase 1：白話版「這章在幹嘛」）
5. 內容講解，依投影片順序分節（Phase 1 建立；Phase 2 修錯；Phase 4 在對應段落旁插圖）
6. 跟前面像的東西怎麼分（Phase 5：只放在較後面章節的講義，比較表 + 一句話判斷準則）
7. 考古題與詳解（Phase 6：該章考古題全收錄，逐步詳解引用前面的教學與錨點）

### 標題層級契約

全課程合併列印時，所有章節固定使用同一套標題層級，不依章節作者或原投影片格式改變：

| Markdown 層級 | 固定用途 | 範例 |
|---|---|---|
| `#` | 一份講義唯一的章名 | `# 第 14 章：簡單線性迴歸` |
| `##` | 講義主區塊 | `## 先備知識`、`## 學習目標`、`## 本章重點一覽`、`## 內容講解`、`## 跟前面像的東西怎麼分`、`## 考古題與詳解` |
| `###` | 正文大節、跨章比較項目、題庫題型範圍 | `### 14.1 簡單線性迴歸模型`、`### 比較 1：斜率 t vs 迴歸 F`、`### 選擇題｜第 1–20 題：基本觀念` |
| `####` | 正文小節、例題、共用題組、單一題號 | `#### 最小平方法`、`#### 題組 14-1：選擇題 64–68 共用資料`、`#### 選擇題 1` |
| `#####` | 單題內固定分區 | 只使用 `##### 題目` 與 `##### 詳解` |

- 不跳級：`##` 之下先有 `###`，需要再細分才使用 `####`；單題的 `#####` 必須隸屬於一個 `#### 選擇題 N` 或 `#### 計算題 N`。
- 不使用 `######`，避免合併列印時字級過小。
- 投影片頁碼放在標題文字尾端即可，不因投影片原本的視覺層級改變 Markdown 層級。
- 跨章比較標題統一寫成 `### 比較 N：主題 A vs 主題 B`。
- 英文考古題的 `MC`、`Multiple Choice`、`Problem`、`Exhibit` 只保留在原題內容；中文導覽標題統一使用「選擇題」「計算題」「題組」。
- 這套層級適用所有既有與未來章節；不得為新章另創同義標題或換一套層級。

考古題固定使用以下結構；有多個題型範圍時可重複 `### 選擇題｜...` 或 `### 計算題｜...`：

```markdown
## 考古題與詳解

### 選擇題｜第 1–20 題：基本觀念

#### 題組 15-1：選擇題 10–14 共用資料

#### 選擇題 1

##### 題目

> English original question and options

##### 詳解

1. **辨認題型：** ...
2. **選方法：** ...
3. **檢查假設：** ...
4. **代入計算／推理：** ...
5. **解讀結論：** ...

### 計算題｜第 1–10 題：完整推論

#### 計算題 1

##### 題目

> English original problem

##### 詳解

1. **辨認題型：** ...
```

`course_2/題型攻略.md`（Phase 8）是唯一獨立於章節講義之外的學習文件：每章一節，題型分類、解題套路、陷阱與注意事項，引用講義錨點不重抄。

## 助教口吻標準

寫給「上課聽不懂、需要有人重講一遍」的新手：

- 先直覺後公式：每個新觀念先用生活或商業情境講一遍「它在解決什麼問題」，再給定義與符號。
- 每個公式的每個符號都用表列解釋（含單位）；不寫「顯然」「易得」。
- 短句、主動語態；一段只講一件事。密集公式前後用白話句緩衝。
- 專有名詞第一次出現：繁中為主、附英文，如 `**卡方檢定(chi-square test)**`。
- 每個方法都要講「什麼時候用、什麼時候不能用、誤用會怎樣」。
- 老師投影片的例題用自己的話重新走一遍解題過程，不逐字照抄。

## VSCode Markdown 相容規則（列印前提）

1. 數學只用 dollar 定界：行內 `$E = mc^2$`、獨立 `$$ \bar{x} = \frac{1}{n}\sum x_i $$`。不用 `\( ... \)` 或 `\[ ... \]`。
2. 一律用 ASCII 括號 `( )`，含中英術語標註：寫 `**平均數(mean)**`，不寫 `**平均數（mean）**`。
3. 每個閉合 `**` 之後、下一個字元之前，手動放一個 ASCII 空格：`**平均數(mean)** 是...`。
4. 圖片一律用相對路徑（`figures/generated/...`），確保合併列印與 VSCode 預覽都能顯示。

標記任何 phase 完成前，搜尋變更過的 Markdown 中的 `\(`、`\[`、`（`、`）`、以及 `**` 閉合後直接接非空格字元的情況；刻意例外需人工確認。

## 公式契約

每個重要公式包含：

- 全課程唯一的錨點 ID（`<a id="formula-..."></a>` 放在公式正前方）
- 公式名稱與用途
- 數學式
- 符號表（意義與單位）
- 假設與適用條件
- 判斷線索：什麼時候用它
- 對照線索：什麼時候不用它、該考慮哪個鄰近公式
- 情境中的解讀
- 可行時附一個量級或單位的合理性檢查

## 定性意義契約（Phase 7）

重要公式、統計量、模型指標與方向性關係不能只說「如何算」或「越大／越小」。在第一次需要的地方，以 `**定性意義：**` 或自然銜接的短段落補足：

- **資料長相**：數值改變時，觀察值的分散、群體重疊、分布相似度、殘差圖案或時間連續性如何改變。
- **訊號與雜訊**：該數值是在量組間差異、組內離散、抽樣不穩定、未解釋誤差、預測不確定性，還是模型與資料的不相容程度。
- **條件與基準**：比較的是誰和誰、固定了哪些條件、基準模型或參考組是什麼。
- **決策語言**：可以合理說什麼，以及不能把它誤說成因果、證明、機率倍數、個體保證或樣本外表現。

優先補強下列常見斷層：

| 數理敘述 | 至少要補出的定性觀念 |
|---|---|
| 標準差、標準誤或區間寬度變大／變小 | 個體離散、估計量重複抽樣的集中程度與參數精確度是三件不同的事 |
| 檢定統計量絕對值或 p 值變化 | 效果相對抽樣雜訊的位置，以及資料和虛無模型的相容程度 |
| $\chi^2$、F、SSE、MSE、$R^2$ 的變化 | 分布不相似、組間分離、組內離散、未解釋誤差、相對基準模型的改善 |
| 斜率、部分斜率、交互作用 | 平均反應如何改變、固定了哪些變數、效果是否依條件改變 |
| logistic 機率、勝算與勝算比 | 基準機率不同時，相同勝算倍數不代表相同機率增量 |
| Durbin-Watson 或自相關 | 相鄰殘差是形成同號長串、近似隨機交錯，還是規律正負交替 |

章末可增加「把數字翻成真實意義」摘要，但不得只靠摘要補救正文；公式或例題附近仍需有局部解讀。

## 考古題詳解契約（Phase 6）

- 該章考古題 PDF 中**每一題**都要收錄，保留題目原文（英文題保留英文，可附一句白話翻譯）。
- 詳解步驟固定：辨認題型 → 選方法（引用 Phase 5 判斷準則）→ 檢查假設 → 代入計算（含單位）→ 解讀結論。
- 詳解必須引用本講義前面教過的說法與公式錨點；不得引入講義沒教過的解法。
- 選擇題每個誘答選項都寫「為什麼會想選它、錯在哪」；不得只標對錯。
- 每題答案獨立驗算（必要時用 Python 重算），驗算方式記入 `course_2/進度.md` 驗收證據。
- 單題標題只寫 `#### 選擇題 N` 或 `#### 計算題 N`；每題恰有一個 `##### 題目` 和一個 `##### 詳解`，而且題目在前、詳解在後。共用資料另用 `#### 題組 ...`，不可取代單題標題。
- 在「選方法」第一個分句先連到本題核心公式／方法；次要診斷、限制或延伸方法放在分號後。合併器依這個順序建立反向題號索引。
- 跨章方法使用明確的相對檔案錨點，例如 `15-multiple-regression.md#formula-...`；不要為了出題章方便而複製同一公式。
- 正典章節不得手寫 `exam-chN-mc-K`、`exam-chN-problem-K`、「對應考古題」或「回看講義」；這些由發布腳本產生，避免正文和題庫失去同步。

## 圖表契約（Phase 4）

- 腳本放 `course_2/figures/scripts/`，輸出放 `course_2/figures/generated/`，分開存放；用 `uv run` 執行。
- 隨機模擬必設種子；記錄重現指令。
- 座標軸與單位都標示；不只靠顏色傳達資訊。
- alt text 寫統計上的重點（該注意什麼），不是畫面描述。
- 標明是觀察資料、模擬資料還是示意圖。
- 全課程統一視覺風格（調色、字型、尺寸）。

## 素材使用界線

投影片、課本（11e，節號與投影片的 14e Metric 可能有差）、考古題都是版權教材，僅供個人讀書用途的轉化整理。查核時以內容對照為準；投影片與課本不一致時以投影片（考試）為準並在講義註明差異。講義不得公開發布或部署為對外網站。

## 完成證據

把某格狀態改成 `完成` 前，在 `course_2/進度.md` 記錄：

- 審閱過的檔案（含投影片頁碼範圍或考古題題號）
- 執行過的驗證（涵蓋檢查、課本核對、驗算、圖表重現）
- 尚存限制
- 完成日期

骨架與 TODO 佔位一律算 `未開始`，不算 `完成`。

## 進度帳本契約

`course_2/進度.md` 是唯一權威進度帳本，包含三個部分：**素材公布紀錄**（哪一天拿到哪些投影片/考古題、解鎖了哪些工作）、**講義進度表**（一列一份講義，欄位為 Phase 1–7）、**驗收證據**。Phase 8（題型攻略）在表格下方獨立記錄狀態。狀態只用 `未開始`、`進行中`、`待驗收`、`完成`、`不適用`（例如 ch7–10 複習講義的 Phase 6）。每次拿到新素材，當天在素材公布紀錄加一行並同步更新受影響的格子。

- 同一份講義後一 phase 需等前一 phase `完成`。
- Phase 5 另需被比較的前面章節講義 Phase 1 已 `完成`。
- Phase 7 在該講義的 Phase 6 為 `完成` 或 `不適用` 後開始。
- Phase 8 需所有適用章節的 Phase 7 皆 `完成`。
- 尚未取得投影片的章節，Phase 1 標 `未開始` 並註明等待素材；素材登記後才解鎖。
- 永不把這些狀態複製進講義檔——帳本與副本會失去同步，而且副本會被印出去。
